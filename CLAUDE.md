# FretBoard — Claude 작업 규칙

## 계획서 작성 규칙

- 계획서(plan file)에는 **리팩토링 후 상세 코드를 절대 작성하지 않는다**
- Task 단위로 "무엇을 바꿀 것인가 + 어떤 파일 + 왜" 수준만 기술
- **현재 코드(Before)는 문제를 명확히 보여주기 위해 포함해도 된다** — 어디가 문제인지 근거로 활용
- 리팩토링 방향(어떻게 바꿀 것인가)은 서술로 기술 가능
- 메서드 구현, 변수명 등 구체적인 구현 세부사항(After 코드)은 제외 — 구현 결정은 tdd-team 에이전트에 위임
- 테스트 시나리오 이름 정도는 허용 (예: `회원가입_후_로그인_성공`)

## TDD 실행

- TDD 실행은 `toby-multi-agent:tdd-team` 스킬에 위임
- 에이전트가 컨텍스트를 보고 직접 구현 결정
